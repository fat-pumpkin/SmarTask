# Changelog

## [2.0.0] - 2026-07-15

### ✨ 优化

- 功能区 Row2 按钮标签外置并排显示，按钮统一方形尺寸，等宽分布占满整行
- 功能区 Row1 统计项（待办/逾期/今日/近期/进度条）调大尺寸，加阴影边框与 hover 效果，醒目直观
- 日历视图任务条加大尺寸并显示任务描述，强化优先级色块区分（左边框 + 背景色 + 优先级图标）
- 设置页全部文案迁移到 i18n（en/zh），About 版本号改为动态读取 manifest.version
- queryTasks 委托 QueryEngine.query，消除重复筛选逻辑
- onload 尊重 indexingEnabled 配置，支持设置页热切换索引开关
- 修复子任务插入：基于父任务原始缩进计算子缩进，插入到父任务子树末尾
- 修复添加任务后输入框跳到页面顶部的 DOM 重排 bug

### 🐛 修复（Obsidian 社区审核违规）

- 修复 QuickCreateModal 与 SmartTaskViewController 中 setTimeout 未清理问题
- 6 处硬编码 hex 颜色改为 CSS 变量（var(--priority-*) / var(--text-error)）
- 4 处 setTimeout magic number 提取为具名常量

## [1.0.5] - 2026-07-13

### 🐛 修复

- 修复移动端输入框太小导致输入内容被遮挡的问题，改为 textarea 支持根据内容自动展开
- 修复"添加"按钮出现在输入框内遮挡输入内容的问题，将按钮移至工具栏同行
- 修复移动端功能栏占用太多空间的问题，改为可折叠模式，移动端默认折叠
- 修复移动端标题与功能按钮分行显示导致空间浪费的问题，合并为同一行
- 修复看板/列表视图中操作按钮（编辑/添加子任务）重叠和挤压文字导致竖排的问题
- 修复看板模式下始终显示待办和已完成两列的问题，根据筛选状态显示对应列
- 修复交错时间线在移动端布局异常的问题，改为单列布局
- 修复代码中直接设置 style 属性的 lint 警告，改用 CSS class 和 setCssStyles
- 移除所有 CSS 中的 !important，改用更高特异性选择器

### ✨ 优化

- 操作按钮统一为 meta-action-btn 样式，移至任务描述下方 meta 行内水平排列
- 看板列在移动端改为纵向堆叠，避免水平滚动
- 卡片网格在移动端改为单列布局
- 日历格子触摸区域优化
